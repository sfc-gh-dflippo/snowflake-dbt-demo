# Snowpark Migration Accelerator: Invalid Access Code error on VDI[¶](#snowpark-migration-accelerator-invalid-access-code-error-on-vdi "Link to this heading")

## I’m getting an access code error using a VDI. What should I do?[¶](#i-m-getting-an-access-code-error-using-a-vdi-what-should-i-do "Link to this heading")

If you receive an “invalid access code” error when trying to use SMA after restarting your Virtual Desktop Infrastructure (VDI) or reconnecting to it, even though you previously activated an access code successfully:

![](../../../../_images/image%28504%29.png)

When using virtualization software, SnowConvert access codes are linked to your computer’s unique serial number. If the virtualization software changes the UUID (Universal Unique Identifier) of the virtual machine, you may encounter activation errors. To resolve this:

1. Ensure your virtualization software maintains a consistent UUID for the virtual machine
2. Note that copying SnowConvert files to another computer will not work unless both computers have identical serial numbers

To reactivate your license, delete the activation files from the following Windows folder:

```
C:\Users\<Your User>\AppData\Roaming\Snowflake Inc
```

Copy

For macOS operating system:

```
/Users/<Your User>/.config/Snowflake Inc
```

Copy

To prevent the error from recurring and avoid having to reactivate the access code, ensure that the UUID configuration is maintained in the virtualized software.
